# Security Policy

## Supported Versions

Currently, only the latest version on the `main` branch is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| older   | :x:                |

## Reporting a Vulnerability

We take the security of HealthML seriously. If you discover a security vulnerability within HealthML, please send an e-mail to the repository owners. All security vulnerabilities will be promptly addressed.

Please do **NOT** report security vulnerabilities via public GitHub issues.

**What to include in your report:**
- Type of vulnerability (e.g., XSS, SQLi, RCE)
- Step-by-step instructions to reproduce the issue
- Potential impact of the vulnerability

We will try to acknowledge your report within 48 hours, and will send a more detailed response within 5 days indicating the next steps in handling your report.
