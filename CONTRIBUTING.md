# Contributing to HealthML

First off, thank you for considering contributing to HealthML! It's people like you that make HealthML such a great tool.

## How Can I Contribute?

### Reporting Bugs

Before creating bug reports, please check the existing issues list as you might find out that you don't need to create one. When you are creating a bug report, please include as many details as possible:

* Use a clear and descriptive title for the issue to identify the problem.
* Describe the exact steps which reproduce the problem in as many details as possible.
* Provide specific examples to demonstrate the steps. Include links to files or GitHub projects, or copy/pasteable snippets, which you use in those examples.
* Describe the behavior you observed after following the steps and point out what exactly is the problem with that behavior.
* Explain which behavior you expected to see instead and why.

### Suggesting Enhancements

Enhancement suggestions are tracked as GitHub issues.

* Use a clear and descriptive title for the issue to identify the suggestion.
* Provide a step-by-step description of the suggested enhancement in as many details as possible.
* Explain why this enhancement would be useful to most HealthML users.

### Pull Requests

* Fill in the required template
* Do not include issue numbers in the PR title
* Include screenshots and animated GIFs in your pull request whenever possible.
* End all files with a newline.
* Follow the existing coding style and architecture.

## Development Setup

1. Fork the repo and create your branch from `main`.
2. Install dependencies for Node backend and React frontend via `npm install`.
3. Install Python dependencies in a virtual environment for the ML service.
4. If you've added code that should be tested, add tests.
5. Ensure the test suite passes (`npm run test` or Playwright E2E tests).

## Code of Conduct

This project and everyone participating in it is governed by the [HealthML Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.
